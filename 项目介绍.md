## 项目定位

Novel Server 是一个多用户在线小说阅读系统。它提供账号登录、书架管理、小说搜索、搜索记录、章节阅读、阅读进度同步、阅读偏好保存、头像/封面上传、管理员用户管理和书城来源管理。NestJS 服务统一使用 `/api` 前缀暴露接口，根路径 `/` 返回内置单页网页端。

## 核心功能

### 用户与认证

- 注册、登录、退出登录。
- 通过用户名和绑定邮箱/手机号重置密码。
- Bearer Token 会话鉴权。
- `POST /api/auth/heartbeat` 心跳保活，网页端约每 30 秒发送一次。
- `POST /api/auth/avatar` 上传用户头像，支持 JPG、PNG、GIF、WEBP、HEIC、HEIF、AVIF，服务端限制 10MB，文件保存在 `uploads` 并通过 `/uploads/...` 访问。
- 用户被禁用或删除后，会话会失效，网页端清除本地登录态并返回登录页。

### 管理员能力

- 查看用户列表和在线状态。
- 启用、禁用普通用户。
- 将普通用户密码重置为初始密码 `guest2026`，并清除该用户会话。
- 删除普通用户，并清除该用户关联会话、书架、进度和偏好数据。
- 禁止禁用、删除或重置管理员账号。
- 管理书城小说网站列表，可启用或停用对应来源；至少保留一个启用来源。

### 小说来源与搜索

- 来源列表、搜索、小说详情、章节列表和章节正文接口都需要登录态。
- 搜索不再由前台展示或选择具体来源；用户搜索只会使用管理员在“书城管理”里启用的小说网站。
- 当前内置适配器为 `quanben.io`，其启用状态由管理员“书城管理”控制。
- 搜索关键词会按当前用户写入搜索记录，支持单条删除和全部清空；每个用户只能看到自己的搜索记录。
- 章节目录和正文按需抓取，正文会缓存到数据库。

### 书架与上传

- 支持从搜索结果加入书架。
- 支持手动添加书籍。
- 支持修改书架条目的自定义标题、封面和排序。
- 支持上传书籍封面，支持 JPG、PNG、GIF、WEBP、HEIC、HEIF、AVIF，服务端限制 10MB，文件保存在 `uploads` 并通过 `/uploads/...` 访问。
- 网页端书架卡片支持轻点封面阅读，长按呼出玻璃操作面板，可更换封面或删除书籍。
- 手机端书架全屏浏览使用原生滚动，确保上下滑动跟手；桌面/触控板滚轮使用累计阈值和冷却，避免滑动跳书。
- 图片封面等比例放大铺满全屏，右下角书名和提示放在模糊玻璃背景框内。

### 阅读体验

- 支持章节阅读、上一章/下一章跳转和目录抽屉。
- 自动保存章节、滚动位置和阅读百分比。
- 阅读页为沉浸模式，隐藏常规顶部和底部导航。
- 点击正文中间区域可呼出/收起浮层控制条。
- 支持字号、夜间模式和护眼模式，这些设置保存到用户偏好。

### 网页端界面

- 登录、菜单抽屉、书城、用户管理和书城管理已改为暗色玻璃视觉。
- 书城页面隐藏具体来源名称，只保留搜索框和搜索按钮；搜索记录以胶囊形式展示在原推荐位，“全部记录”按钮位于最左侧，展开后可查看全部记录并一键删除；桌面和手机端都保留结果区域，避免页面空旷。
- 书架仍保留沉浸式大封面浏览。
- 阅读器保留纸张阅读主题。
- loading、toast、禁用/删除账号提示、删除确认等交互统一在内置网页端处理；上传接口遇到非 JSON 错误响应时会显示可读错误，不再暴露 `JSON Parse error`。

## 存储状态

`StoreModule` 通过 `STORE_DRIVER` 切换存储：

- 未设置 `STORE_DRIVER=mysql`：使用 `InMemoryStore`，适合开发/演示。
- 设置 `STORE_DRIVER=mysql`：使用 `MysqlStore`，云机当前就是该模式。

MySQL 启动时会创建缺失表，并执行轻量迁移。2026-07-01 部署后已确认 `users.avatar_url` 字段存在。

云机环境变量：

| 变量 | 说明 | 示例 |
| --- | --- | --- |
| `STORE_DRIVER` | 存储驱动 | `mysql` |
| `DB_HOST` | MySQL 主机 | `127.0.0.1` |
| `DB_PORT` | MySQL 端口 | `3306` |
| `DB_USER` | MySQL 用户名 | `novel_server` |
| `DB_PASSWORD` | MySQL 密码 | 不在文档记录 |
| `DB_NAME` | MySQL 数据库名 | `novel_server` |
| `PORT` | HTTP 服务端口 | `3000` |
| `ADMIN_PASSWORD` | 首次创建默认管理员时可使用的密码变量 | 可选 |

默认管理员只会在不存在管理员账号时创建：

```text
username: admin
password: ADMIN_PASSWORD 或 Admin12345
```

## 数据模型概览

- `User`：用户账号、角色、状态、头像、登录时间。
- `Session`：登录会话 token 和在线状态时间。

- `ReadingProgress`：章节、滚动位置和阅读百分比。
- `SearchHistoryItem`：用户搜索关键词、搜索次数和最近搜索时间。
- `UserPreferences`：书架、底部导航和阅读器主题设置。
- `UploadedFile`：上传文件元数据。
