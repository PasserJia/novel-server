## 项目定位

Novel Server 是一个部署在腾讯云服务器上的多用户在线小说阅读系统。系统提供账号登录、书架管理、小说搜索、章节阅读、阅读进度同步、界面偏好保存和管理员用户管理等功能；根路径 `/` 返回内置网页端，接口统一使用 `/api` 前缀。

## 核心功能

### 用户与认证

- 用户注册、登录、退出登录。
- 通过用户名和绑定联系方式重置密码。
- 使用 Bearer Token 会话进行接口鉴权。
- 用户被禁用或删除后，相关会话会失效；网页端会弹窗提示“本用户已被禁用或删除”，点击确定后清除本地登录态并返回登录页。

### 管理员能力

- 管理员可查看用户列表。
- 管理员可启用或禁用普通用户。
- 管理员可删除普通用户，删除后该用户会被强制登出。
- 禁止禁用或删除管理员账号，避免系统失去管理入口。

### 小说来源与搜索

- 当前接入 `quanben.io` 作为小说来源。
- 支持通过关键字搜索小说。
- 支持抓取小说目录和章节正文。
- 章节正文按需抓取并缓存，减少重复访问源站。

### 个人书架

- 用户可将搜索结果加入个人书架。
- 支持手动添加书籍。
- 支持修改书架条目的自定义标题、封面和排序。
- 支持删除书架条目。
- 支持上传自定义封面，上传文件通过 `/uploads` 静态路径访问。

### 阅读体验

- 支持章节阅读、上一章/下一章跳转。
- 支持目录抽屉。
- 自动保存阅读进度，包括章节、滚动位置和百分比。
- 阅读页不展示书架、书城、用户这组三个 tab。
- 手机端非阅读页面的三个 tab 固定在底部（胶囊样式，适配底部安全区）。
- 进入阅读为全屏沉浸模式：桌面端与手机端都会隐藏顶部 Logo 栏，正文铺满整屏（手机端用 100dvh 适配浏览器地址栏，不保留外层边框）。
- 阅读控制条默认隐藏；点击正文中间区域可呼出/收起，控制条为浮于正文之上的毛玻璃浮层。
- 呼出的控制条包含：顶部「退出（返回书架）、阅读进度、设置、目录」，底部「上一章、下一章」；点击退出会先保存当前阅读位置再返回书架。
- 桌面端宽屏时控制条按钮对齐居中正文列；划选/复制正文时不会误触发控制条收起。
- 夜间模式、护眼模式下控制条浮层会同步变色；支持字号、夜间模式和护眼模式等阅读偏好。

### 界面反馈

- 系统内按钮点击/接口请求会显示统一 loading 弹窗，请求结束后自动关闭。
- loading 弹窗为暖纸色渐变卡片、深色半透明遮罩、青金双色旋转环与衬线文字，带轻微缩放淡入，和「水墨纸张」主题统一。
- 后台状态检查、滚动自动保存和页面关闭前保存进度使用静默请求，不打断用户阅读。
- 登录成功不再弹出提示，登录后直接进入书架；操作失败、删除、封面更新等仍有 toast 提示。
- 用户被禁用或删除时弹出居中提示弹窗，点击确定后清除本地登录态并返回登录页。

### 界面视觉

- 登录页与内部页采用「水墨纸张 / Ink & Paper」主题：暖纸色背景、衬线标题与正文（书名、章节正文使用宋体衬线字体栈，纯系统字体、无外部字体依赖，规避国内访问 Google Fonts 的风险）、以青绿（青）为主的强调色，延续黑白青的系统基调。
- 登录页带柔和的青/金/印泥红径向光晕与纸纹网格背景；卡片、书架行、封面、阅读器均有分层阴影与渐变描边。
- 内置克制的微交互（错峰淡入、按钮按下回弹、悬停轻抬、抽屉滑入、toast 滑入、封面悬停微翻等），并遵循系统的「减少动态效果」（prefers-reduced-motion）偏好，必要时自动降级为无动画。

### 偏好设置

- 书架主题、底部导航主题和阅读器主题统一保存在用户偏好中。
- 云机实际运行环境使用 MySQL 保存偏好数据。

### 模块划分

```text
src/
  main.ts                         应用启动入口，全局 API 前缀、校验管道、静态上传目录
  app.module.ts                   根模块
  modules/
    auth/                         注册、登录、会话鉴权、角色守卫
    admin/                        管理员用户管理
    bookshelf/                    个人书架、手动添加、封面上传
    novels/                       小说搜索结果入库、目录、章节正文读取
    preferences/                  用户阅读和界面偏好
    reading-progress/             阅读进度读取与保存
    sources/                      小说源列表和 quanben.io 适配器
    web/                          内置单页网页端
  store/
    models.ts                     核心数据模型类型
    passwords.ts                  密码哈希与校验
    in-memory.store.ts            内存存储实现
    mysql.store.ts                MySQL 存储实现，包含建表和数据映射
    store.module.ts               存储实现切换入口
```

### 存储状态

项目通过 `StoreModule` 使用同一个注入 token 暴露存储能力：

- 未设置 `STORE_DRIVER=mysql` 时使用 `InMemoryStore`。
- 设置 `STORE_DRIVER=mysql` 时使用 `MysqlStore`。

云机环境已确认设置 `STORE_DRIVER=mysql`，实际使用 `MysqlStore`。MySQL 存储会在服务启动时创建缺失的数据表，并在不存在管理员账号时创建默认管理员。

## 数据模型概览

- `User`：用户账号、角色、状态、登录时间。
- `Session`：登录会话 token。
- `Novel`：小说基础信息和来源标识。
- `Chapter`：章节目录、正文缓存和源站地址。
- `BookshelfItem`：用户书架条目。
- `ReadingProgress`：章节、滚动位置和阅读百分比。
- `UserPreferences`：书架、底部导航和阅读器主题设置。
- `UploadedFile`：上传文件元数据。
